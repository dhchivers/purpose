# Vision Agent Documentation

## Overview

The Vision Agent is an AI-powered system that helps users create compelling vision statements for their future (5, 10, or 15 years ahead). It synthesizes user inputs about meaningful change, influence scale, and their role into three differentiated vision statement options, each presenting a distinct perspective on the user's desired future impact.

**Key Features:**
- Timeframe selection with career stage guidance
- Contextual integration with user's purpose and values
- Four guided input questions
- AI generation of three distinct vision styles
- Selection and editing capabilities
- Complete session history for regeneration

**AI Model:** OpenAI GPT-4o (via GeminiService wrapper)

---

## Vision Agent Philosophy

### What is a Vision Statement?

A vision statement is a **present-tense description of changed conditions** that exist in the future because the user acted on their purpose. It is **not** a goal or task list.

**Core Principles:**
1. **Present Tense as If Already True** - "Systems operate differently" not "I will change systems"
2. **Changed Conditions, Not Tasks** - Describe the world that exists, not what you did
3. **Scale-Appropriate** - Reflects whether impact is on individuals, organizations, institutions, or systems
4. **Purpose-Aligned** - Extensions of the user's core purpose statement
5. **Values-Constrained** - Guided and limited by the user's core values
6. **No Metrics** - Avoids specific numbers, milestones, or KPIs

### Three Vision Styles

The Vision Agent generates three differentiated options to help users explore different framings:

| Style | Focus | Characteristics |
|-------|-------|-----------------|
| **Clear & Strategic** | Observable outcomes | Straightforward, concrete, tangible shifts |
| **Systems & Institutional** | Structural changes | How systems and organizations function differently |
| **Meaning & Human Impact** | Human experience | How people's lives and relationships are different |

## Data Structures

### 1. VisionOption

```dart
class VisionOption {
  final String label;      // "Clear & Strategic", "Systems & Institutional", "Meaning & Human Impact"
  final String statement;  // The vision statement text
}
```

**Purpose:** Represents a single AI-generated vision statement option.

**Serialization:** JSON via build_runner

---

### 2. InfluenceScale (Enum)

```dart
enum InfluenceScale {
  @JsonValue('individuals')
  individuals,        // Direct impact on people through relationships, coaching

  @JsonValue('organizations')
  organizations,      // Impact on teams, companies, or groups

  @JsonValue('institutions')
  institutions,       // Impact on schools, governments, major organizations

  @JsonValue('systems')
  systems,           // Impact on interconnected systems, industries, societal structures
}
```

**Purpose:** Defines the scope and scale of the user's intended influence.

**UI Display:**
- `individuals` → "Individuals"
- `organizations` → "Organizations"
- `institutions` → "Institutions"
- `systems` → "Broader Systems"

**JSON Serialization:** Uses `@JsonValue` annotations for Firestore compatibility.

---

### 3. VisionCreationSession

```dart
class VisionCreationSession {
  // Identification
  final String id;              // Unique session ID
  final String userId;          // User who created this session
  
  // Timestamps
  final DateTime startedAt;     // When session began
  final DateTime? completedAt;  // When vision was finalized (null if in progress)
  
  // USER INPUTS (Step 1-4)
  final int? timeframeYears;           // 5, 10, or 15 years
  final String? meaningfulChange;      // Response to "What meaningful change exists?"
  final InfluenceScale? influenceScale; // individuals/organizations/institutions/systems
  final String? roleDescription;       // What role user plays in creating change
  
  // CONTEXT (Loaded from User Profile)
  final String? purposeStatement;      // User's purpose statement
  final List<String>? coreValues;      // User's core values (from user_values collection)
  
  // AI RESULTS (Step 5)
  final List<VisionOption>? visionOptions;  // 3 options generated by AI
  final int? selectedOptionIndex;           // Which option user selected (0, 1, or 2)
  final String? customStatement;            // If user edited the selected option
}
```

**Purpose:** Tracks the complete vision creation journey with all inputs and AI results.

**Lifecycle:**
1. Created when user starts vision creation flow
2. Updated progressively as user completes each step
3. AI results added after step 4
4. Marked complete when user finalizes vision
5. Persisted for regeneration and history

**Firestore Collection:** `vision_creation_sessions`

**Firestore Operations:**
- `saveVisionCreationSession()` - Create/update session
- `getVisionCreationSession()` - Retrieve by ID for regeneration

---

### 4. UserVision

```dart
class UserVision {
  final String id;                    // Unique vision ID
  final String userId;                // User who owns this vision
  final int timeframeYears;           // 5, 10, or 15 years
  final String visionStatement;       // The finalized vision statement
  final String? sessionId;            // Reference to VisionCreationSession
  final DateTime createdAt;           // When vision was first created
  final DateTime updatedAt;           // Last modification date
}
```

**Purpose:** Stores the user's finalized vision statement.

**Firestore Collection:** `user_visions`

**Firestore Operations:**
- `saveUserVision()` - Create new vision (increments user.visionCount, updates user.vision)
- `getUserVision()` - Get most recent vision for user
- `userVisionStream()` - Real-time vision updates
- `updateUserVision()` - Update existing vision (updates user.vision field)
- `deleteUserVision()` - Delete vision (decrements visionCount, sets user.vision to next or null)

**User Profile Integration:** The `user.vision` field is updated to match the vision statement for dashboard display.

## Vision Creation Flow

The vision creation process consists of 7 steps across two phases: **User Input (Steps 1-4)** and **AI Generation & Selection (Steps 5-7)**.

### Phase 1: User Input Collection

#### Step 1: Timeframe Selection

**UI Component:** Three large selection cards

**Options:**
- **5 Years** - "School age and early career"
  - Guidance: "Focus on skill development, foundational experiences, and early impact."
- **10 Years** - "Early to mid-career"
  - Guidance: "Focus on establishing expertise, building influence, and scaling impact."
- **15 Years** - "Mid to late career"
  - Guidance: "Focus on systemic change, legacy building, and transformative impact."

**Data Captured:** `timeframeYears` (int)

**Validation:** Required - cannot proceed without selection

**Session Update:** `session.copyWith(timeframeYears: selected)`

---

#### Step 2: Meaningful Change Question

**Question Presented:**
```
In [X] years, what meaningful change exists because you acted on your purpose of [Purpose]?
```

**Context Display:**
- User's purpose statement shown in a highlighted card
- Reminds user of their purpose while thinking about future impact

**UI Component:** Multi-line text field (6 lines)

**Guidance:** "Focus on changed conditions, not your activities. What's different in the world? How do people/systems operate differently?"

**Data Captured:** `meaningfulChange` (string)

**Validation:** Required - cannot be empty

**Session Update:** `session.copyWith(meaningfulChange: text)`

---

#### Step 3: Influence Scale Selection

**Question Presented:**
```
In the [X]-year timeframe, are you primarily influencing individuals, organizations, institutions, or broader systems?
```

**Options:** Four selection cards with icons and descriptions

1. **Individuals** (👤 person_outline)
   - "Direct impact on people through relationships, coaching, or one-on-one interactions."

2. **Organizations** (🏢 business_outlined)
   - "Impact on teams, companies, or groups through leadership, culture, or collaboration."

3. **Institutions** (🏛️ account_balance_outlined)
   - "Impact on established entities like schools, governments, or major organizations."

4. **Broader Systems** (🔗 hub_outlined)
   - "Impact on interconnected systems, industries, or societal structures."

**Data Captured:** `influenceScale` (InfluenceScale enum)

**Validation:** Required - must select one option

**Session Update:** `session.copyWith(influenceScale: selected)`

---

#### Step 4: Role Description

**Question Presented:**
```
What role are you playing in bringing about this change?
```

**UI Component:** Multi-line text field (6 lines)

**Example Roles Provided:**
- "Catalyst - Initiating and sparking change"
- "Builder - Creating new structures or systems"
- "Connector - Bridging people, ideas, or resources"
- "Educator - Teaching, training, or developing others"
- "Advocate - Championing causes or perspectives"
- "Innovator - Pioneering new approaches or solutions"

**Data Captured:** `roleDescription` (string)

**Validation:** Required - cannot be empty

**Session Update:** `session.copyWith(roleDescription: text)`

---

### Phase 2: AI Generation & Selection

#### Step 5: AI Vision Generation

**Trigger:** User completes Step 4 and clicks "Next"

**Process:**
1. Display loading indicator
2. Call `geminiService.generateVisionStatements()`
3. Pass all collected data plus context
4. Receive 3 vision options
5. Save session with `visionOptions`
6. Navigate to selection step

**Loading State:** Full-screen CircularProgressIndicator with "Generating your vision options..."

**AI Call Parameters:**
```dart
await geminiService.generateVisionStatements(
  timeframeYears: session.timeframeYears!,
  purposeStatement: session.purposeStatement ?? '',
  coreValues: session.coreValues ?? [],
  meaningfulChange: session.meaningfulChange!,
  influenceScale: session.influenceScale!.name,
  roleDescription: session.roleDescription!,
);
```

**Success:** Returns `List<Map<String, dynamic>>` with 3 vision options

**Error Handling:** Fallback to generic statements if AI call fails

---

#### Step 6: Vision Option Selection

**UI Display:** Three vision option cards

**Card Content:**
- **Label** (bold, primary color) - "Clear & Strategic", "Systems & Institutional", or "Meaning & Human Impact"
- **Statement** (body text, graphite color) - The full vision statement
- **Selection Indicator** - Blue border and checkmark when selected

**Interaction:** Tap card to select option

**Data Captured:** `selectedOptionIndex` (0, 1, or 2)

**Validation:** Required - must select one option

**Session Update:** `session.copyWith(selectedOptionIndex: index)`

---

#### Step 7: Optional Editing & Finalization

**UI Components:**

1. **Selected Vision Display Card** (highlighted with primary tint)
   - Shows label of selected option
   - Shows statement in read-only mode by default
   - "Edit" button to enable editing

2. **Editing Mode:**
   - TextField with 5 lines
   - Pre-populated with selected statement
   - "Cancel" button to revert
   - Changes saved to `customStatement`

3. **Complete Button:**
   - Creates `UserVision` object
   - Saves to Firestore
   - Updates user profile
   - Navigates to dashboard

**Finalization Process:**

**Session Completion:**

## AI Agent: Vision Synthesis Agent

### Agent Configuration

**Model:** `gpt-4o` (OpenAI)  
**Temperature:** `0.8` (Higher creativity for diverse options)  
**Max Tokens:** `1000`  
**Response Format:** JSON array

---

### Exact AI Prompt

```
You are a Vision Synthesis Agent.

You have received the following context:

**Timeframe:** [timeframeYears] years
**Purpose Statement:** "[purposeStatement]"
**Core Values:** [coreValues joined with commas]

**User Responses:**
1. Meaningful Change: "[meaningfulChange]"
2. Influence Scale: [influenceScale]
3. Role: "[roleDescription]"

Your task:
1. Identify consistent patterns in desired impact
2. Confirm alignment with core values
3. Detect unrealistic or goal-based framing and correct it
4. Generate exactly 3 differentiated Vision statement options

Each Vision statement must:
- Be written in present tense as if already true
- Describe changed conditions, not tasks or goals
- Reflect the scale ([influenceScale]) and domain
- Remain aligned with Purpose
- Remain constrained by Values
- Avoid listing metrics or milestones
- Paint a picture of the world that exists because they lived their purpose

Generate 3 options with these specific styles:

**Option 1 — Clear & Strategic**
Straightforward, concrete description of changed conditions. Focus on observable outcomes and tangible shifts.

**Option 2 — Systems & Institutional**
Emphasize structural changes, institutional transformation, or systemic patterns. Focus on how systems and organizations function differently.

**Option 3 — Meaning & Human Impact Integrated**
Connect changed conditions to deeper meaning and human experience. Focus on how people's lives and relationships are different.

Return ONLY a JSON array with this exact structure:
[
  {
    "label": "Clear & Strategic",
    "statement": "<clear strategic vision statement here>"
  },
  {
    "label": "Systems & Institutional",
    "statement": "<systems and institutional vision statement here>"
  },
  {
    "label": "Meaning & Human Impact",
    "statement": "<meaning and human impact vision statement here>"
  }
]

No additional text, just the JSON array.
```

---

### AI Prompt Breakdown

**Context Section:**
- **Timeframe** - Influences scope and ambition of vision
- **Purpose Statement** - Core identity that vision must align with
- **Core Values** - Constraints and guidance for vision content
- **User Responses** - Raw data about desired change, scale, and role

**Task Instructions:**
1. **Pattern Identification** - Find themes in user's inputs
2. **Alignment Confirmation** - Ensure vision fits purpose and values
3. **Unrealistic Framing Detection** - Convert goals to changed conditions
4. **Generation** - Create 3 distinct options

**Vision Statement Requirements (6 rules):**
- Present tense as if already true
- Changed conditions, not tasks
- Reflect scale and domain
- Aligned with purpose
- Constrained by values
- No metrics or milestones

**Style Specifications:**
- **Option 1** - Concrete, observable, tangible
- **Option 2** - Structural, systemic, organizational
- **Option 3** - Meaning-focused, human experience, relational

### UserVision Operations

#### saveUserVision()

**Purpose:** Create a new user vision and update user profile

**Parameters:**
- `UserVision vision`

**Process:**
1. Convert vision to JSON
2. Convert `DateTime` fields to `Timestamp`
3. Set document in `user_visions` collection
4. **Update user profile:**
   - Set `user.vision` field to vision statement
   - Increment `user.visionCount`
   - Update `user.updatedAt` timestamp

**Code:**
```dart
final data = vision.toJson();
data['createdAt'] = Timestamp.fromDate(vision.createdAt);
data['updatedAt'] = Timestamp.fromDate(vision.updatedAt);

await _userVisionsCollection.doc(vision.id).set(data);

// Update user document
final userDoc = _usersCollection.doc(vision.userId);
await userDoc.update({
  'vision': vision.visionStatement,
  'visionCount': FieldValue.increment(1),
  'updatedAt': FieldValue.serverTimestamp(),
});
```

**Why Update user.vision?** Dashboard displays `user.vision` field for quick access without additional queries.

---

#### getUserVision()

**Purpose:** Get user's most recent vision statement

**Parameters:**
- `String userId`

**Returns:** `UserVision?` (null if no visions)

**Process:**
1. Query `user_visions` where `userId` matches
2. Order by `updatedAt` descending
3. Limit to 1 result
4. Convert `Timestamp` to `DateTime`
5. Parse to `UserVision`

**Code:**
```dart
final querySnapshot = await _userVisionsCollection
    .where('userId', isEqualTo: userId)
    .orderBy('updatedAt', descending: true)
    .limit(1)
    .get();

if (querySnapshot.docs.isEmpty) return null;

final doc = querySnapshot.docs.first;
final data = doc.data() as Map<String, dynamic>;

// Convert Timestamps
if (data['createdAt'] is Timestamp) {
  data['createdAt'] = (data['createdAt'] as Timestamp).toDate().toIso8601String();
}
if (data['updatedAt'] is Timestamp) {
  data['updatedAt'] = (data['updatedAt'] as Timestamp).toDate().toIso8601String();
}

return UserVision.fromJson(data);
```

#### updateUserVision()

**Purpose:** Update an existing vision statement

**Parameters:**
- `UserVision vision`

**Process:**
1. Convert vision to JSON
2. Update `updatedAt` to current timestamp
3. Set document in `user_visions` collection
4. **Update user profile:**
   - Set `user.vision` to updated statement
   - Update `user.updatedAt` timestamp

**Code:**
```dart

data['createdAt'] = Timestamp.fromDate(vision.createdAt);
data['updatedAt'] = Timestamp.fromDate(DateTime.now());

await _userVisionsCollection.doc(vision.id).set(data);

// Update user document

await userDoc.update({
  'vision': vision.visionStatement,
  'updatedAt': FieldValue.serverTimestamp(),
});
```

---

#### deleteUserVision()

**Purpose:** Delete a vision and update user profile

**Parameters:**
- `String visionId`
- `String userId`

**Process:**
1. Delete document from `user_visions` collection
2. Query for next most recent vision
3. **Update user profile:**
   - If remaining visions exist: Set `user.vision` to next vision
   - If no remaining visions: Set `user.vision` to null
   - Decrement `user.visionCount`
   - Update `user.updatedAt` timestamp

**Code:**
```dart
await _userVisionsCollection.doc(visionId).delete();

// Get next most recent vision
final remainingVisions = await _userVisionsCollection
    .where('userId', isEqualTo: userId)
    .orderBy('updatedAt', descending: true)
    .limit(1)
    .get();

final userDoc = _usersCollection.doc(userId);

if (remainingVisions.docs.isEmpty) {
  // No more visions
  await userDoc.update({
    'vision': null,
    'visionCount': FieldValue.increment(-1),
    'updatedAt': FieldValue.serverTimestamp(),
  });
} else {
  // Update to next vision
  final nextVisionData = remainingVisions.docs.first.data() as Map<String, dynamic>;
  await userDoc.update({
    'vision': nextVisionData['visionStatement'],
    'visionCount': FieldValue.increment(-1),
    'updatedAt': FieldValue.serverTimestamp(),
  });
}
```

## Vision Regeneration Process

Users can regenerate their vision with modified inputs on the Vision Page.

### Regeneration Flow

1. **User Opens Vision Page**
   - Vision and session data loaded
   - "Edit & Regenerate" button displayed

2. **User Clicks "Edit & Regenerate"**
   - All question cards become editable
   - Pre-populated with current values
   - User modifies any/all inputs

3. **User Clicks "Regenerate Vision"**
   - Loading state displayed
   - AI called with updated inputs:
     ```dart
     await geminiService.generateVisionStatements(
       timeframeYears: editedTimeframe,
       purposeStatement: session.purposeStatement,
       coreValues: session.coreValues,
       meaningfulChange: editedMeaningfulChange,
       influenceScale: editedInfluenceScale.name,
       roleDescription: editedRoleDescription,
     );
     ```

4. **AI Returns New Options**
   - Session updated with new `visionOptions`
   - Saved to Firestore

5. **Selection Dialog Appears**
   - 3 new options displayed as cards
   - User selects preferred option
   - Click "Apply"

6. **Vision Updated**
   - New statement saved to `UserVision`
   - User profile `vision` field updated
   - Caches invalidated
   - Success message displayed
   - Edit mode cancelled

### Regeneration vs. New Creation

| Aspect | Regeneration | New Creation |
|--------|-------------|--------------|
| Session | Updates existing | Creates new |
| Context | Preserves purpose/values | Loads from profile |
| Inputs | Pre-populated | Empty |
| History | Adds to session | New session |
| Vision ID | Same | New |

## Integration with Other Modules

### Purpose Module
- **Input:** User's purpose statement is required context for vision generation
- **Flow:** Users typically complete Purpose before Vision
- **Data:** `user.purpose` field loaded into `VisionCreationSession.purposeStatement`

### Values Module
- **Input:** User's core values are required context for vision generation
- **Flow:** Users typically complete Values before Vision
- **Data:** `user_values` collection queried, statements loaded into `VisionCreationSession.coreValues`

### Dashboard
- **Display:** Vision card shows timeframe and statement
- **Navigation:** 
  - If vision exists → Vision Page
  - If no vision → Vision Creation Flow
- **Updates:** Real-time via `currentUserProvider` invalidation

### Mission Module (Future)
- **Input:** Vision will inform mission planning
- **Relationship:** Mission breaks down vision into actionable pathways

### Goals Module (Future)
- **Input:** Vision sets the long-term destination for goals
- **Relationship:** Goals are concrete milestones toward vision

## Technical Implementation Notes

### State Management
- **Provider:** Riverpod
- **Pattern:** FutureProvider for async data loading
- **Cache:** Invalidation on updates for immediate UI refresh

### Error Handling
- AI service failures → Fallback to generic vision statements
- Network errors → User-friendly error messages with retry options
- Validation → Step-by-step validation prevents incomplete data

### Performance
- Lazy loading of session data (only when needed)
- Efficient Firestore queries with ordering and limits
- Single vision per user (most recent) for dashboard simplicity

### Security
- Firestore rules ensure users can only access their own visions
- Session data tied to authenticated user ID
- No sensitive data in vision statements

### Web Compatibility
- Timestamp conversion for all DateTime fields
- JSON serialization via build_runner
- Responsive UI adapts to screen sizes

## Summary

The Vision Agent is a sophisticated AI-powered system that helps users articulate their desired future impact through a guided 7-step process. By combining user inputs about meaningful change, influence scale, and role with their existing purpose and values, the AI generates three distinct vision statement options—Clear & Strategic, Systems & Institutional, and Meaning & Human Impact—allowing users to explore different framings of their future aspirations.

**Core Innovation:** Present-tense vision statements that describe changed conditions (not goals) give users a concrete picture of the world they're creating, making abstract future aspirations feel tangible and actionable.

**Architecture Strengths:**
- Complete session history for regeneration
- Purpose and values integration
- Three distinct AI perspectives
- Flexible editing and refinement
- Seamless dashboard integration

The Vision Agent transforms abstract future thinking into concrete vision statements that guide long-term decision-making and align with users' core identity and values.
